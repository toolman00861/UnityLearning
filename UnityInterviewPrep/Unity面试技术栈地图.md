# Unity面试技术栈地图（2026求职向）

## 一、企业最常见的硬性要求（必须过关）

### 1) C#与计算机基础
- C#基础与面向对象（类、接口、泛型、委托、事件、反射）
- 常用数据结构与算法（数组、链表、哈希、树、排序、查找）
- 设计模式基础（单例、观察者、工厂、状态）
- 多线程与异步基础（Task、线程安全、主线程调度）

### 2) Unity核心能力
- 生命周期与组件系统（MonoBehaviour、GameObject、Prefab）
- 2D/3D物理系统（Rigidbody、Collider、Raycast）
- UI系统（UGUI、Canvas、事件系统）
- 动画系统（Animator、BlendTree、状态机）
- 场景与资源组织（Scene、Prefab Variant、依赖关系）

### 3) 工程化与性能优化
- Profiler与Frame Debugger定位性能瓶颈
- GC Alloc控制与内存优化（对象池、减少临时分配）
- Draw Call、Batching、贴图与材质优化
- 打包与平台配置（Android/iOS/PC基本构建流程）

## 二、当前市场高频“加分项”

### 1) 资源与热更新体系
- Addressables资源分组、远程加载、版本更新
- 常见热更新思路（Lua体系或C#热更体系，如HybridCLR）
- AB包与资源依赖、冗余和卸载策略

### 2) 渲染与图形方向
- URP基础与渲染流程理解
- Shader Graph与基础HLSL
- 常见后处理与性能折中方案

### 3) 联机与客户端协作
- 与服务端协议联调经验（消息结构、状态同步）
- 网络同步基础（状态同步/帧同步概念）
- 断线重连、重试、资源下载容错思路

### 4) 工具链与交付能力
- Git协作、分支策略、代码规范
- 自动化构建与测试意识（至少会配置基础流程）
- 可展示作品集（完整可运行Demo + 技术复盘）

## 三、按岗位方向分层准备

### 初级Unity开发（0-2年）
- 重点：C#基础 + Unity常用系统 + 两个完整Demo
- 目标：能独立完成常规模块并修复常见Bug

### 中级Unity开发（2-5年）
- 重点：性能优化 + 架构拆分 + 资源体系 + 联机协作
- 目标：可负责一个系统模块从设计到交付

### 专项方向（可选）
- 客户端主程倾向：架构与性能深度
- 技术美术倾向：渲染、Shader、工具制作
- XR倾向：AR Foundation / OpenXR / 多端适配优化

## 四、你可以直接使用的面试准备顺序

1. 先补C#基础和数据结构，保证手写题不卡壳
2. 再补Unity核心模块，形成“能讲清楚”的工程经验
3. 然后专项突破一个方向（性能或渲染或热更新）
4. 最后整理作品集与项目复盘，做面试表达训练

## 五、市场信息参考（用于确认方向）

- BOSS直聘职位信息中，Unity岗位常见关键词集中在：C#、数据结构、算法、客户端逻辑开发、性能优化、协作沟通。
- 招聘侧对资源管理、渲染优化、跨端适配、问题排查能力要求持续上升。
- Unity官方2025路线与行业内容显示，稳定性、性能、跨平台与XR覆盖仍是持续投入重点，具备相关经验有利于求职竞争力。
