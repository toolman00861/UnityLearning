# 虚拟内存与内存对齐 (Virtual Memory & Memory Alignment)

## 虚拟内存 (Virtual Memory)

### 概念
虚拟内存是计算机系统内存管理的一种技术。它为每个进程提供了一个一致的、私有的地址空间，使得应用程序认为自己拥有连续可用的内存（即虚拟地址空间），而实际上，它通常被分隔成多个物理内存碎片，还有部分暂时存储在外部磁盘存储器上，在需要时进行数据交换。

### 核心机制
1.  **虚拟地址 (Virtual Address)**：程序使用的地址。
2.  **物理地址 (Physical Address)**：RAM 实际的地址。
3.  **页表 (Page Table)**：
    *   **映射**：记录虚拟页号（VPN）到物理页框号（PFN）的映射关系。
    *   **权限控制**：如只读、可执行等。
4.  **MMU (Memory Management Unit)**：CPU 中的硬件单元，负责将虚拟地址转换为物理地址。
5.  **缺页中断 (Page Fault)**：
    *   当程序访问一个尚未加载到物理内存的虚拟页时，CPU 会触发缺页异常。
    *   操作系统捕获该异常，从磁盘加载对应的数据页到物理内存，并更新页表。
6.  **TLB (Translation Lookaside Buffer)**：
    *   页表的缓存，位于 CPU 内部，加速地址转换。

### 优势
1.  **隔离性**：每个进程拥有独立的地址空间，互不干扰，增强系统稳定性。
2.  **安全性**：通过页表权限位控制访问权限（如防止代码段被修改）。
3.  **内存利用率**：
    *   **按需加载**：只加载当前需要的页面。
    *   **共享库**：多个进程可以共享只读的代码段（如 libc.so），节省物理内存。
4.  **扩展性**：使得程序可以使用比实际物理内存更大的地址空间（利用 swap 分区）。

## 内存对齐 (Memory Alignment)

### 概念
内存对齐是指数据在内存中存储时的起始地址必须是该数据类型大小的整数倍（或者遵循特定的对齐规则）。

### 为什么要内存对齐？
1.  **性能原因 (Performance)**：
    *   CPU 访问内存通常是按块（如 4 字节、8 字节、16 字节甚至 64 字节的 Cache Line）读取的。
    *   **对齐访问**：如果一个 `int`（4 字节）存储在地址 0x00，CPU 一次读取即可获取完整数据。
    *   **非对齐访问**：如果该 `int` 存储在地址 0x01，跨越了两个内存块（0x00-0x03 和 0x04-0x07），CPU 可能需要两次内存访问，并进行拼接操作，导致性能下降。
    *   某些架构（如 ARM, MIPS）对于非对齐访问甚至会直接抛出硬件异常（Bus Error）。
2.  **平台移植性**：不同硬件平台对存储空间的处理上有很大的不同。

### 结构体对齐规则 (Struct Alignment)
在 C/C++ 中，结构体的大小并不简单的等于各成员大小之和，需遵循以下规则：
1.  **自身对齐**：结构体每个成员的起始地址必须是其自身大小（或 `#pragma pack` 指定值中较小者）的整数倍。
2.  **整体对齐**：结构体的总大小必须是其最大成员大小（或 `#pragma pack` 指定值中较小者）的整数倍。
3.  **填充 (Padding)**：为了满足上述规则，编译器会在成员之间或结构体末尾插入填充字节。

#### 示例
```cpp
// 假设是 64 位系统，默认对齐
struct A {
    char a;     // 1 byte (Offset: 0)
    // Padding: 3 bytes
    int b;      // 4 bytes (Offset: 4)
    short c;    // 2 bytes (Offset: 8)
    // Padding: 6 bytes (为了让总大小是 8 的倍数)
    long long d;// 8 bytes (Offset: 16)
}; 
// sizeof(A) = 24 bytes (1+3+4+2+6+8)

// 优化后的顺序
struct B {
    long long d;// 8 bytes (Offset: 0)
    int b;      // 4 bytes (Offset: 8)
    short c;    // 2 bytes (Offset: 12)
    char a;     // 1 byte  (Offset: 14)
    // Padding: 1 byte (为了让总大小是 8 的倍数)
};
// sizeof(B) = 16 bytes (8+4+2+1+1)
```

### 常用指令
*   `alignof(T)` / `_Alignof(T)`：获取类型的对齐要求。
*   `#pragma pack(n)`：指定结构体的对齐字节数（n 通常为 1, 2, 4, 8, 16）。
    *   `pack(1)` 表示紧凑排列，无填充，常用于网络协议包头定义，但访问效率低。
