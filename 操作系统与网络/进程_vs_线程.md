# 进程 vs 线程 (Process vs Thread)

## 核心区别

进程（Process）和线程（Thread）是操作系统中两个核心概念，它们的主要区别在于**资源分配**和**调度执行**。

| 特性 | 进程 (Process) | 线程 (Thread) |
| :--- | :--- | :--- |
| **定义** | 资源分配的最小单位 | CPU 调度的最小单位 |
| **资源拥有** | 拥有独立的地址空间、文件句柄、堆、数据段等资源 | 共享所属进程的资源（堆、全局变量、代码段），但拥有独立的栈（Stack）和寄存器状态 |
| **开销** | 创建、销毁、切换开销大（需要通过内核态，涉及页表切换等） | 开销小（轻量级，同一个进程内的线程切换只需保存/恢复少量寄存器） |
| **通信** | 进程间通信（IPC）复杂（管道、消息队列、共享内存、Socket 等） | 线程间通信简单（直接读写共享内存），但需注意同步互斥（锁、信号量） |
| **健壮性** | 独立性强，一个进程崩溃通常不影响其他进程 | 一个线程崩溃（如非法内存访问）可能导致整个进程崩溃 |
| **并发性** | 多进程并发，适合多核 CPU | 多线程并发，适合 I/O 密集型或计算密集型任务 |

## 详细对比

### 1. 资源隔离与共享
*   **进程**：每个进程都有自己独立的虚拟地址空间。这意味着进程 A 无法直接访问进程 B 的内存，必须通过操作系统提供的 IPC 机制。
*   **线程**：同一进程内的所有线程共享进程的内存空间（代码段、数据段、堆）。
    *   **共享**：堆（Heap）、全局变量、打开的文件、静态数据。
    *   **私有**：栈（Stack，用于函数调用和局部变量）、程序计数器（PC，指示下一条指令）、寄存器（保存当前执行状态）。

### 2. 上下文切换 (Context Switch)
*   **进程切换**：
    1.  切换页目录（Page Directory）以使用新的地址空间（最耗时，会导致 TLB 缓存失效）。
    2.  切换内核栈和硬件上下文。
*   **线程切换**：
    1.  如果是在同一进程内切换，不涉及虚拟地址空间的切换，TLB 依然有效。
    2.  只需切换少量的寄存器和线程私有栈。

### 3. 通信机制
*   **IPC (Inter-Process Communication)**：
    *   管道 (Pipe) / 命名管道 (Named Pipe)
    *   消息队列 (Message Queue)
    *   共享内存 (Shared Memory) + 信号量 (Semaphore)
    *   套接字 (Socket)
    *   信号 (Signal)
*   **线程同步与通信**：
    *   互斥锁 (Mutex)
    *   读写锁 (Read-Write Lock)
    *   条件变量 (Condition Variable)
    *   自旋锁 (Spinlock)
    *   信号量 (Semaphore)

## 适用场景

*   **多进程**：
    *   需要高稳定性和安全性的场景（如 Chrome 浏览器的每个标签页是一个进程，崩溃不影响主程序）。
    *   分布式系统。
*   **多线程**：
    *   需要频繁创建和销毁的任务。
    *   需要大量计算且数据共享频繁的场景（如视频编码、游戏引擎中的物理模拟）。
    *   GUI 程序（通常主线程负责 UI，工作线程负责耗时操作）。

## 总结
*   **进程**是操作系统分配资源的容器。
*   **线程**是 CPU 执行任务的流水线。
*   一个进程至少包含一个线程（主线程）。
