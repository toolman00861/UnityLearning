# 02. 顶点与片元着色器入门：Hello World

我们将从零开始编写一个最简单的 Shader，并在过程中理解每一行代码的作用。

## 目标
创建一个 Shader，使物体显示为单一颜色，并且这个颜色可以在 Unity 面板中调节。

## 第一步：搭建骨架

清空你的 Shader 文件，输入以下基础结构：

```shader
Shader "MyShader/02_SimpleColor" { // 定义Shader路径和名字
    Properties {
        // 这里定义属性，稍后我们会用到
    }
    
    SubShader {
        Tags { "RenderType"="Opaque" } // 告诉Unity这是一个不透明物体
        
        Pass {
            CGPROGRAM // 开始HLSL代码
            
            // 声明顶点和片元函数名
            #pragma vertex vert
            #pragma fragment frag
            
            #include "UnityCG.cginc" // 引入Unity常用函数库
            
            // 下面我们将定义结构体和函数...
            
            ENDCG // 结束HLSL代码
        }
    }
}
```

## 第二步：定义数据结构

在 Shader 中，数据需要在不同阶段之间传递。我们需要定义两个结构体：

1.  **appdata**：定义**顶点着色器**接收的数据（从模型网格 Mesh 中读取）。
2.  **v2f**：定义**顶点着色器**输出给**片元着色器**的数据（Vertex to Fragment）。

在 `#include "UnityCG.cginc"` 下方添加：

```shader
            // 1. 从模型获取的数据结构 (Application Data)
            struct appdata {
                float4 vertex : POSITION; // 告诉Unity把模型顶点坐标填充到这个变量
                // float3 normal : NORMAL; // 如果需要法线，可以取消注释
                // float2 uv : TEXCOORD0;  // 如果需要UV，可以取消注释
            };

            // 2. 传递给片元着色器的数据结构 (Vertex to Fragment)
            struct v2f {
                float4 pos : SV_POSITION; // 裁剪空间坐标 (必须包含！)
            };
```

*   `: POSITION`、`: SV_POSITION` 叫做 **语义 (Semantics)**。它们告诉 GPU 这个变量代表什么含义。
*   `float4` 是一个四维向量 (x, y, z, w)。位置通常用 float4 表示。

## 第三步：编写顶点着色器 (Vertex Shader)

顶点着色器的核心任务：把模型空间的顶点坐标转换到裁剪空间。

在结构体下方添加：

```shader
            v2f vert (appdata v) {
                v2f o; // 定义输出结构体
                
                // 核心计算：模型空间 -> 裁剪空间
                // UnityObjectToClipPos 是 UnityCG.cginc 提供的内置函数
                o.pos = UnityObjectToClipPos(v.vertex);
                
                return o; // 返回计算后的数据
            }
```

*   `UnityObjectToClipPos(v.vertex)` 等价于矩阵乘法 `mul(UNITY_MATRIX_MVP, v.vertex)`，是把 3D 世界的点“拍扁”到 2D 屏幕上的关键步骤。

## 第四步：编写片元着色器 (Fragment Shader)

片元着色器的任务：决定每个像素的颜色。

在 `vert` 函数下方添加：

```shader
            fixed4 frag (v2f i) : SV_Target {
                // 直接返回红色 (R=1, G=0, B=0, A=1)
                return fixed4(1, 0, 0, 1); 
            }
```

*   `fixed4`：低精度的 float4，用于颜色足够了（范围 -2 ~ 2，精度 1/256）。
*   `: SV_Target`：告诉 GPU 把这个函数的返回值输出到屏幕（帧缓存）。

---

## 第五步：让颜色可调 (Properties)

现在 Shader 只能显示红色。我们来添加一个属性，让它可以在材质面板修改。

1.  **修改 Properties 块**：

```shader
    Properties {
        _MainColor ("My Color", Color) = (1, 1, 1, 1) // 定义一个名为 _MainColor 的属性
    }
```

2.  **在 CGPROGRAM 中声明变量**：
    ShaderLab 属性 (`_MainColor`) 和 HLSL 代码是分离的。要在代码里用这个属性，必须定义一个同名变量。

    在 `#include "UnityCG.cginc"` 下方添加：

```shader
            fixed4 _MainColor; // 变量名必须与 Properties 中的一致
```

3.  **修改片元着色器使用该变量**：

```shader
            fixed4 frag (v2f i) : SV_Target {
                return _MainColor; // 返回面板设置的颜色
            }
```

---

## 完整代码 (Copy & Paste)

```shader
Shader "MyShader/02_SimpleColor" {
    Properties {
        _MainColor ("My Color", Color) = (1, 1, 1, 1)
    }
    SubShader {
        Tags { "RenderType"="Opaque" }
        LOD 100

        Pass {
            CGPROGRAM
            #pragma vertex vert
            #pragma fragment frag
            #include "UnityCG.cginc"

            struct appdata {
                float4 vertex : POSITION;
            };

            struct v2f {
                float4 pos : SV_POSITION;
            };

            fixed4 _MainColor; // 声明变量

            v2f vert (appdata v) {
                v2f o;
                o.pos = UnityObjectToClipPos(v.vertex);
                return o;
            }

            fixed4 frag (v2f i) : SV_Target {
                return _MainColor; // 使用变量
            }
            ENDCG
        }
    }
}
```

## 练习
1.  保存代码，在 Unity 中创建一个 Material 使用这个 Shader。
2.  尝试修改 `_MainColor`，观察物体颜色变化。
3.  尝试让片元着色器返回 `fixed4(0, 1, 0, 1)` (绿色)，忽略 `_MainColor`。
4.  **进阶挑战**：尝试让颜色随时间变化。提示：使用内置变量 `_Time.y` (代表游戏运行时间，秒)。
    *   修改为 `return _MainColor * abs(sin(_Time.y));` 看看会发生什么？
