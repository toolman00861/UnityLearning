# 图论 (Graph Theory) - 最短路径与并查集

图论是游戏开发中路径规划、地图生成、社交关系分析等场景的基础。

## 1. 核心概念

*   **图 (Graph)**: 由顶点 (Vertex) 和边 (Edge) 组成。
*   **有向图 (Directed)** vs **无向图 (Undirected)**: 边是否有方向。
*   **权重 (Weight)**: 边的代价（如距离、消耗时间）。
*   **度 (Degree)**: 连接到顶点的边的数量（入度/出度）。
*   **存储**: 邻接矩阵 (Adjacency Matrix) 适合稠密图，邻接表 (Adjacency List) 适合稀疏图。

## 2. 最短路径算法 (Shortest Path)

### 2.1 Dijkstra 算法
**适用场景**: 单源最短路径，边权非负。
**核心思想**: 贪心策略，每次扩展距离源点最近的未访问节点。
**数据结构**: 优先队列 (Priority Queue / Min-Heap)。

**步骤**:
1. 初始化 `dist` 数组，源点为 0，其余为无穷大。
2. 将源点加入优先队列。
3. 循环直到队列为空：
    a. 取出当前距离最小的节点 `u`。
    b. 遍历 `u` 的所有邻居 `v`。
    c. **松弛操作 (Relax)**: 如果 `dist[u] + weight(u, v) < dist[v]`，更新 `dist[v]` 并将 `v` 加入队列。

**代码片段 (伪代码)**:
```csharp
PriorityQueue<int, int> pq = new PriorityQueue<int, int>();
pq.Enqueue(startNode, 0);

while (pq.Count > 0) {
    int u = pq.Dequeue();
    if (visited[u]) continue;
    visited[u] = true;

    foreach (var neighbor in adj[u]) {
        int v = neighbor.to;
        int weight = neighbor.weight;
        if (dist[u] + weight < dist[v]) {
            dist[v] = dist[u] + weight;
            pq.Enqueue(v, dist[v]);
        }
    }
}
```

### 2.2 A* (A-Star) 算法
**适用场景**: 游戏中的寻路（Pathfinding），特别是有明确目标点的情况。
**核心思想**: 启发式搜索 (Heuristic Search)。
**公式**: `f(n) = g(n) + h(n)`
*   `g(n)`: 从起点到当前节点的实际代价。
*   `h(n)`: 从当前节点到终点的预估代价（启发函数）。

**常见启发函数**:
*   **曼哈顿距离 (Manhattan)**: `|x1 - x2| + |y1 - y2|` (只能上下左右移动)
*   **欧几里得距离 (Euclidean)**: `sqrt((x1 - x2)^2 + (y1 - y2)^2)` (任意方向移动)
*   **切比雪夫距离 (Chebyshev)**: `max(|x1 - x2|, |y1 - y2|)` (八方向移动)

**与 Dijkstra 的区别**: A* 利用 `h(n)` 引导搜索方向，通常比 Dijkstra 更快找到目标，但不一定能找到最短路径（除非 `h(n)` 满足一致性条件）。

## 3. 并查集 (Union-Find / Disjoint Set Union - DSU)

**适用场景**: 动态连通性问题（如判断网络中两点是否连通、最小生成树 Kruskal 算法）。
**核心操作**:
1.  **Find**: 查找元素所属集合的代表元素（根节点）。
2.  **Union**: 合并两个集合。

**优化**:
1.  **路径压缩 (Path Compression)**: 在 Find 过程中，将路径上的节点直接指向根节点，扁平化树结构。
2.  **按秩合并 (Union by Rank)**: 将小树合并到大树上，避免树过深。

**代码实现**:
```csharp
class UnionFind {
    private int[] parent;
    private int[] rank;

    public UnionFind(int size) {
        parent = new int[size];
        rank = new int[size];
        for (int i = 0; i < size; i++) {
            parent[i] = i; // 初始时每个节点是自己的父节点
            rank[i] = 1;
        }
    }

    public int Find(int x) {
        if (parent[x] != x) {
            parent[x] = Find(parent[x]); // 路径压缩
        }
        return parent[x];
    }

    public void Union(int x, int y) {
        int rootX = Find(x);
        int rootY = Find(y);

        if (rootX != rootY) {
            // 按秩合并
            if (rank[rootX] > rank[rootY]) {
                parent[rootY] = rootX;
            } else if (rank[rootX] < rank[rootY]) {
                parent[rootX] = rootY;
            } else {
                parent[rootY] = rootX;
                rank[rootX]++;
            }
        }
    }
    
    public bool Connected(int x, int y) {
        return Find(x) == Find(y);
    }
}
```

## 4. 常见应用

1.  **最小生成树 (Minimum Spanning Tree)**: 连接所有节点且总权重最小。
    *   **Kruskal**: 基于边，使用并查集。
    *   **Prim**: 基于点，类似 Dijkstra。
2.  **拓扑排序 (Topological Sort)**: 解决依赖关系（如任务前置条件、编译顺序）。使用入度表和队列。
3.  **洪水填充 (Flood Fill)**: 类似于 BFS/DFS，用于区域着色、扫雷等。

## 5. 总结

*   **Dijkstra**: 适用于求单源最短路径，非负权图。
*   **A***: 适用于有明确终点的寻路，效率高。
*   **并查集**: 适用于动态连通性问题，实现简单且高效。
