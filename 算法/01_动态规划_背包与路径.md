# 动态规划 (Dynamic Programming) - 游戏开发与面试精要

动态规划（DP）是解决最优化问题的核心算法，也是大厂笔试中的常客。其核心思想是将复杂问题分解为子问题，并存储子问题的解以避免重复计算。

## 1. 核心概念

*   **状态定义 (State)**: 用变量描述问题的子结构（如 `dp[i]` 表示前 `i` 个物品的最大价值）。
*   **状态转移方程 (Transition)**: 描述子问题之间的关系（如 `dp[i] = max(dp[i-1], dp[i-2] + value)`）。
*   **初始条件 (Base Case)**: 最小子问题的解。
*   **无后效性**: 未来的决策不会改变过去的状态。

## 2. 背包问题 (Knapsack Problem)

背包问题是资源分配问题的原型，在游戏中常用于装备栏管理、技能点分配等场景。

### 2.1 0/1 背包问题
每个物品只能选择 0 次或 1 次。

**问题描述**: 有 N 件物品和一个容量为 W 的背包。第 i 件物品的重量是 w[i]，价值是 v[i]。求不超过背包容量的最大价值。

**状态定义**: `dp[i][j]` 表示前 i 件物品放入容量为 j 的背包可以获得的最大价值。

**转移方程**:
```csharp
// 如果不选第 i 件物品
dp[i][j] = dp[i-1][j]; 
// 如果选第 i 件物品 (前提是能放得下: j >= w[i])
dp[i][j] = Math.Max(dp[i-1][j], dp[i-1][j - w[i]] + v[i]);
```

**空间优化 (滚动数组)**:
由于 `dp[i]` 只依赖于 `dp[i-1]`，可以将二维数组压缩为一维。
**注意**: 内层循环必须**从大到小**遍历，防止通过当前状态更新当前状态（即防止一个物品被多次放入）。

```csharp
int Knapsack01(int[] weights, int[] values, int capacity) {
    int n = weights.Length;
    int[] dp = new int[capacity + 1];

    for (int i = 0; i < n; i++) {
        // 必须倒序，保证 dp[j-weights[i]] 使用的是上一行的状态
        for (int j = capacity; j >= weights[i]; j--) {
            dp[j] = Math.Max(dp[j], dp[j - weights[i]] + values[i]);
        }
    }
    return dp[capacity];
}
```

### 2.2 完全背包问题
每种物品有无限个。

**转移方程**: 与 0/1 背包类似，但因为可以重复选，所以内层循环**从小到大**遍历。

```csharp
// 完全背包：正序遍历
for (int i = 0; i < n; i++) {
    for (int j = weights[i]; j <= capacity; j++) {
        dp[j] = Math.Max(dp[j], dp[j - weights[i]] + values[i]);
    }
}
```

## 3. 路径问题 (Path Problems)

路径问题通常涉及网格（Grid）中的移动，常用于计算最小代价、路径计数等。

### 3.1 最小路径和 (Minimum Path Sum)
**题目**: 给定一个 m x n 的网格，每个单元格包含一个非负整数。找出一条从左上角到右下角的路径，使得路径上的数字总和最小。每次只能向下或向右移动。

**状态定义**: `dp[i][j]` 表示到达 `(i, j)` 的最小路径和。

**转移方程**:
```csharp
dp[i][j] = grid[i][j] + Math.Min(dp[i-1][j], dp[i][j-1]);
```

**代码实现**:
```csharp
public int MinPathSum(int[][] grid) {
    int m = grid.Length;
    int n = grid[0].Length;
    int[,] dp = new int[m, n];

    dp[0, 0] = grid[0][0];

    // 初始化第一列
    for (int i = 1; i < m; i++) dp[i, 0] = dp[i - 1, 0] + grid[i][0];
    // 初始化第一行
    for (int j = 1; j < n; j++) dp[0, j] = dp[0, j - 1] + grid[0][j];

    for (int i = 1; i < m; i++) {
        for (int j = 1; j < n; j++) {
            dp[i, j] = Math.Min(dp[i - 1, j], dp[i, j - 1]) + grid[i][j];
        }
    }
    return dp[m - 1, n - 1];
}
```

### 3.2 不同路径 (Unique Paths)
**题目**: 机器人从左上角到右下角有多少种不同的路径？

**转移方程**:
```csharp
dp[i][j] = dp[i-1][j] + dp[i][j-1]; // 到达当前格子的路径数 = 上方格子路径数 + 左方格子路径数
```

## 4. 面试常见变体

1.  **打家劫舍 (House Robber)**: 不能偷相邻的房子。 -> `dp[i] = max(dp[i-1], dp[i-2] + nums[i])`
2.  **爬楼梯 (Climbing Stairs)**: 每次爬 1 或 2 阶。 -> `dp[i] = dp[i-1] + dp[i-2]` (斐波那契数列)
3.  **最长递增子序列 (LIS)**: `dp[i]` 表示以 `nums[i]` 结尾的最长递增子序列长度。

## 5. 总结

*   **识别 DP**: 问题包含“最大/最小”、“方案数”、“可行性”等关键词，且具有重叠子问题。
*   **解题步骤**: 
    1. 定义 `dp` 数组含义。
    2. 找出状态转移方程。
    3. 确定 Base Case。
    4. 考虑空间优化（如果只依赖前几个状态）。
