# 渲染管线与核心概念详解

本文档详细解释图形学渲染流程中的核心数据与状态概念，帮助理解 GPU 如何将模型数据转化为屏幕上的像素。

---

## 1. 模型数据 (Mesh Data)
**定义**：模型数据是构建 3D 物体的基础几何信息，通常存储在 **Mesh**（网格）对象中。在渲染管线的**顶点处理阶段**（Vertex Processing），GPU 会读取这些数据。

### 核心分量
*   **顶点位置 (Vertex Position)**: 
    *   **含义**: 定义了物体在**模型空间**（Local Space）中的形状。每个顶点是一个三维坐标 $(x, y, z)$。
    *   **作用**: 决定了物体的几何形态（是球体、立方体还是角色）。
*   **UV (Texture Coordinates)**: 
    *   **含义**: 二维坐标 $(u, v)$，范围通常是 $[0, 1]$。它定义了 2D 纹理贴图如何映射到 3D 模型表面。
    *   **作用**: 就像把一张包装纸（纹理）包裹在礼物盒（模型）上，UV 告诉 GPU 哪个像素贴在哪个顶点上。
*   **法线 (Normal)**: 
    *   **含义**: 每个顶点都有一个垂直于表面的方向向量。
    *   **作用**: **决定光照效果**。光线照射到物体上时，法线决定了光的反射方向和强度（漫反射、高光）。如果没有法线，物体看起来就是平的，没有立体感。
*   **切线 (Tangent)**: 
    *   **含义**: 垂直于法线的向量，通常与纹理的 U 方向一致。
    *   **作用**: 主要用于**法线贴图 (Normal Mapping)** 技术。它和法线、副切线（Binormal/Bitangent）一起构建了“切线空间”，让法线贴图能正确地模拟表面的凹凸细节。

## 2. 变换矩阵 (Transformation Matrices)
**定义**：数学矩阵，用于将顶点坐标从一个空间转换到另一个空间。
**核心作用**: **告诉 GPU 物体在哪里、朝向哪里、有多大，以及摄像机在哪里、怎么看。**

### MVP 变换流程
这是图形学中最经典的变换链路：
1.  **模型矩阵 (Model Matrix)**: 
    *   **变换**: 模型空间 $\rightarrow$ 世界空间 (World Space)。
    *   **作用**: 决定物体在场景中的**位置 (Translation)、旋转 (Rotation)、缩放 (Scale)**。
2.  **观察矩阵 (View Matrix)**: 
    *   **变换**: 世界空间 $\rightarrow$ 观察空间 (View/Camera Space)。
    *   **作用**: 将整个世界移动和旋转，使得**摄像机位于原点**，且看向 Z 轴（通常是 -Z 或 +Z）。相当于“把世界搬到相机面前”。
3.  **投影矩阵 (Projection Matrix)**: 
    *   **变换**: 观察空间 $\rightarrow$ 裁剪空间 (Clip Space)。
    *   **作用**: 
        *   定义摄像机的**视锥体**（Frustum），即能看到的范围。
        *   决定是**透视投影**（近大远小）还是**正交投影**（无近大远小）。
        *   将 3D 坐标压缩到一个标准化的立方体范围内（CVV）。

## 3. 粗粒度剔除 (Culling)
**定义**：在渲染管线早期（通常在顶点着色器之前或之后，但在光栅化之前），快速排除掉那些**绝对看不见**的物体，避免浪费 GPU 算力。

### 视锥体剔除 (Frustum Culling)
*   **原理**: 摄像机的视野是一个平截头体（Frustum）。如果一个物体的**包围盒 (Bounding Box)** 完全在视锥体外面，那么这个物体就完全不可见。
*   **粒度**: **物体级别 (Per-Object)**。CPU 会判断每个物体的包围盒是否与视锥体相交。
*   **作用**: 最基础且最重要的优化手段。如果不做这个，GPU 就要处理整个场景几百万个顶点，即使它们在相机背后。
*   **区别**: 
    *   **视锥体剔除**: 剔除**视野范围外**的物体（CPU 端通常由引擎自动完成）。
    *   **遮挡剔除 (Occlusion Culling)**: 剔除**被其他物体挡住**的物体（需要额外计算或预处理）。

## 4. 渲染状态 (Render State)
**定义**：GPU 是一个状态机。在绘制一个物体之前，必须设置好所有的“开关”和“参数”。这些配置的集合就是渲染状态。

### 关键状态
*   **Shader (着色器)**: GPU 执行的代码（顶点程序 + 片元程序），定义了“怎么画”。
*   **材质参数 (Material Properties)**: Shader 用到的变量，如颜色、光泽度、金属度等。
*   **纹理贴图 (Textures)**: 采样用的图片数据。
*   **混合模式 (Blend Mode)**: 
    *   决定当前画出来的像素颜色，如何与屏幕上已经存在的颜色混合。
    *   例如：`SrcAlpha + OneMinusSrcAlpha` 是典型的透明混合。
*   **深度测试 (Z-Test) & 深度写入 (Z-Write)**: 
    *   决定当前像素是否被前面的物体遮挡，以及是否更新深度缓冲区。
*   **剔除模式 (Cull Mode)**: 
    *   决定剔除正面还是背面（通常剔除背面 Back，双面渲染则为 Off）。

**Draw Call 与 SetPass Call**:
*   改变渲染状态（如切换材质、切换 Shader）通常会触发 **SetPass Call**，这是一个昂贵的 CPU 操作。因此，**合批 (Batching)** 的核心目的就是减少渲染状态的切换。

## 5. 光栅化阶段 (Rasterization)
**定义**：渲染管线的中间阶段，位于顶点处理之后，片元处理之前。
**核心任务**: **将几何图形（三角形）转化为像素（片元）。**

### 流程解析
1.  **图元组装 (Primitive Assembly)**: 将顶点连成三角形。
2.  **三角形遍历 (Triangle Traversal)**: 
    *   判断屏幕上的哪些像素被这个三角形覆盖。
    *   如果一个像素中心在三角形内，就会生成一个**片元 (Fragment)**。
3.  **插值 (Interpolation)**: 
    *   利用**重心坐标 (Barycentric Coordinates)**，根据三个顶点的数据（颜色、UV、法线等），计算出该片元位置的中间值。
    *   例如：三角形三个角分别是红、绿、蓝，光栅化后，中间的片元就会呈现出平滑过渡的渐变色。
4.  **输出**: 生成一系列片元，每个片元包含屏幕坐标、深度值 (Z) 以及插值后的数据，送入**片元着色器 (Fragment Shader)** 进行着色计算。

### 总结图谱

| 概念 | 阶段 | 作用 | 形象比喻 |
| :--- | :--- | :--- | :--- |
| **模型数据** | 输入 | 提供形状和表面信息 | 骨架和皮肤 |
| **变换矩阵** | 顶点处理 | 定位和投影 | 摆姿势、架相机 |
| **视锥体剔除**| 预处理 | 去掉看不见的东西 | 把相机背后的东西扔掉 |
| **光栅化** | 中间阶段 | 几何转像素 | 把矢量图画成位图 |
| **渲染状态** | 全局配置 | 定义绘制规则 | 调色盘和画笔设置 |
