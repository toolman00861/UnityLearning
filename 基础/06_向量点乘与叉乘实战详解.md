# 游戏开发数学基础：向量点乘与叉乘详解

在游戏开发中，**向量 (Vector)** 是最基础的数学工具。其中，**点乘 (Dot Product)** 和 **叉乘 (Cross Product)** 是判断物体位置关系、计算光照、处理物理碰撞的核心算法。

---

## 1. 点乘 (Dot Product)

### 1.1 数学定义与几何意义
点乘的结果是一个**标量 (Scalar)**（即一个单纯的数字，没有方向）。

*   **公式**: $\vec{A} \cdot \vec{B} = |\vec{A}| |\vec{B}| \cos\theta$
    *   $|\vec{A}|$ 是向量 A 的长度（模）。
    *   $\theta$ 是两个向量之间的夹角。
*   **Unity API**: `float dot = Vector3.Dot(a, b);`

### 1.2 核心几何意义
1.  **投影 (Projection)**: 
    *   如果 $\vec{B}$ 是单位向量（长度为1），那么 $\vec{A} \cdot \vec{B}$ 的结果就是 **$\vec{A}$ 在 $\vec{B}$ 方向上的投影长度**。
2.  **夹角判定 (Angle)**:
    *   **结果 > 0**: 夹角为锐角 (0° ~ 90°)，方向基本相同（**前方**）。
    *   **结果 = 0**: 夹角为 90°，互相垂直。
    *   **结果 < 0**: 夹角为钝角 (90° ~ 180°)，方向基本相反（**后方**）。

### 1.3 实战案例：视野判定 (FOV)

**需求**: 判断敌人是否在玩家的视野范围内（例如前方 60 度扇形区域）。

```csharp
using UnityEngine;

public class FieldOfViewCheck : MonoBehaviour
{
    public Transform target; // 敌人
    public float viewAngle = 60f; // 视野角度 (总角度)
    public float viewRadius = 10f; // 视野距离

    void Update()
    {
        if (target == null) return;

        Vector3 dirToTarget = target.position - transform.position; // 目标方向向量
        
        // 1. 距离检查 (这是最廉价的检查，先做)
        // 优化：使用 sqrMagnitude 避免开方运算
        if (dirToTarget.sqrMagnitude > viewRadius * viewRadius) return;

        // 2. 角度检查 (使用点乘)
        // 必须将方向向量归一化，否则点乘结果包含长度信息
        Vector3 forward = transform.forward; // 玩家正前方 (已归一化)
        Vector3 dirNormalized = dirToTarget.normalized; // 目标方向归一化

        float dotResult = Vector3.Dot(forward, dirNormalized);
        
        // 反余弦 Acos 算出角度 (弧度 -> 角度)
        float angle = Mathf.Acos(dotResult) * Mathf.Rad2Deg;

        // 判断角度是否在视野的一半范围内 (因为 dot 算的是中心线两边的夹角)
        if (angle < viewAngle / 2f)
        {
            Debug.Log("敌人在视野内！");
        }
        
        // 【优化技巧】：直接比对 dot 值，避免 Acos (昂贵三角函数)
        // cos(30°) ≈ 0.866
        // if (dotResult > Mathf.Cos(viewAngle / 2f * Mathf.Deg2Rad)) { ... }
    }
}
```

### 1.4 实战案例：光照计算 (Lambert Diffuse)
在 Shader 中，漫反射光照强度取决于 **光线方向** 与 **表面法线** 的夹角。
*   **公式**: `Diffuse = max(0, dot(Normal, LightDir))`
*   当光线垂直射入表面（夹角 0°，dot=1）时，最亮。
*   当光线与表面平行或从背面射入（夹角 >= 90°，dot<=0）时，无光照（变黑）。

## 2. 叉乘 (Cross Product)

### 2.1 数学定义与几何意义
叉乘的结果是一个**向量 (Vector)**，该向量垂直于 A 和 B 构成的平面。

*   **公式**: $\vec{A} \times \vec{B} = \vec{C}$
*   **方向**: 符合**左手定则** (Unity 使用左手坐标系)。
    *   伸出左手，大拇指指向 A，食指指向 B，中指指向的方向就是 C (叉乘结果)。
*   **Unity API**: `Vector3 cross = Vector3.Cross(a, b);`

### 2.2 核心几何意义
1.  **法线 (Normal)**: 
    *   计算三角形或平面的法线向量。
2.  **左右判定**:
    *   通过结果向量的 Y 轴分量（在 2D 平面 X-Z 上），可以判断 B 在 A 的左侧还是右侧。

### 2.3 实战案例：左右方向判定

**需求**: 赛车游戏中，判断下一个检查点是在车的左边还是右边，从而决定自动转向或 UI 提示。

```csharp
using UnityEngine;

public class LeftRightCheck : MonoBehaviour
{
    public Transform checkpoint;

    void Update()
    {
        Vector3 forward = transform.forward; // 车头朝向
        Vector3 dirToCheckpoint = checkpoint.position - transform.position; // 检查点方向

        // 计算叉乘
        Vector3 crossResult = Vector3.Cross(forward, dirToCheckpoint);

        // 在 Unity (左手坐标系, Y轴向上) 中：
        // 如果 crossResult.y > 0，说明 B 在 A 的右侧
        // 如果 crossResult.y < 0，说明 B 在 A 的左侧
        // 如果 crossResult.y = 0，说明共线 (正前或正后)

        if (crossResult.y > 0)
        {
            Debug.Log("向右转 ->");
        }
        else if (crossResult.y < 0)
        {
            Debug.Log("<- 向左转");
        }
    }
}
```

### 2.4 实战案例：程序化生成 Mesh (Procedural Mesh)
在生成自定义地形或几何体时，需要计算每个顶点的法线以正确渲染光照。
*   **步骤**:
    1.  获取三角形的三个顶点 $P_1, P_2, P_3$。
    2.  计算两条边向量: $V_1 = P_2 - P_1$, $V_2 = P_3 - P_1$。
    3.  叉乘求法线: $Normal = Vector3.Cross(V_1, V_2).normalized$。
    4.  赋值给 `mesh.normals`。

## 3. 总结对比

| 特性 | 点乘 (Dot Product) | 叉乘 (Cross Product) |
| :--- | :--- | :--- |
| **结果类型** | **标量 (float)** | **向量 (Vector3)** |
| **几何意义** | 投影长度、夹角余弦值 | 垂直于两向量平面的法向量 |
| **核心用途** | 1. 视野检测 (FOV)<br>2. 前后判断<br>3. 光照计算 (漫反射)<br>4. 投影计算 | 1. 左右判断<br>2. 计算面法线<br>3. 物理力矩计算<br>4. 确定旋转轴 |
| **Unity API** | `Vector3.Dot(a, b)` | `Vector3.Cross(a, b)` |
| **口诀** | **点乘判前后，叉乘分左右** | |
