# 03. 算法与数据结构：从原理到 Unity 工程实战

在 Unity 开发中，算法和数据结构决定了系统的性能上限。

## 1. 数组 (Array) vs 链表 (LinkedList)

### 1.1 内存视角
*   **数组**：**连续内存块**。
    *   **优点**：CPU 缓存友好（预取机制生效），随机访问 O(1)。
    *   **缺点**：插入/删除需要移动元素 O(N)，扩容代价大（申请新空间+拷贝）。
*   **链表**：**离散内存块**（节点通过指针相连）。
    *   **优点**：插入/删除 O(1)（前提是已知节点引用），无需扩容。
    *   **缺点**：CPU 缓存极其不友好（节点在堆上随机分布），随机访问 O(N)，额外指针空间开销。

### 1.2 Unity 实战建议
*   **99% 的情况下优先使用 `List<T>` (动态数组)**。
*   **避免在 `Update` 中频繁操作 `List` 的增删**（尤其是 `Insert` 和 `RemoveAt`）。如果必须频繁增删，考虑：
    1.  **倒序删除**：`for (int i = list.Count - 1; i >= 0; i--)`，O(1) 删除尾部，无需移动元素。
    2.  **交换删除 (Swap-Remove)**：如果不关心顺序，把要删除的元素和最后一个元素交换，然后移除最后一个。O(1)。

```csharp
void FastRemoveAt(List<int> list, int index) {
    int lastIndex = list.Count - 1;
    list[index] = list[lastIndex]; // 覆盖当前元素
    list.RemoveAt(lastIndex); // 移除末尾
}
```

## 2. 哈希表 (Dictionary / HashSet)

### 2.1 底层原理
*   **哈希函数 (Hash Function)**：将 Key 转换为整数索引。
*   **桶 (Buckets)**：哈希值对数组长度取模，决定存放位置。
*   **冲突解决 (Collision Resolution)**：
    *   **拉链法 (Chaining)**：桶里存链表（.NET 核心实现）。当冲突严重时，退化为 O(N)。
*   **扩容 (Rehashing)**：当 `Count >= Capacity` 时，数组翻倍，所有元素重新计算哈希值放入新位置。**极度消耗性能**。

### 2.2 性能陷阱与优化
1.  **预设容量**：如果在循环中添加大量元素，务必使用构造函数 `new Dictionary<int, string>(1000)` 指定初始容量，避免多次扩容。
2.  **避免使用 Enum 作为 Key**：
    *   旧版 .NET/Unity (Mono) 中，Enum 作为 Key 会导致**装箱**（每次 `TryGetValue` 都会产生 GC Alloc）。
    *   **解决**：实现 `IEqualityComparer<T>` 接口，或强制转为 `int` 作为 Key。
3.  **结构体 Key**：必须重写 `GetHashCode` 和 `Equals`，否则默认实现使用反射，极其缓慢且产生 GC。

## 3. 栈 (Stack) 与 队列 (Queue)

### 3.1 典型应用
*   **栈 (LIFO)**：
    *   **UI 弹窗管理**：打开新弹窗压栈，关闭弹窗出栈。
    *   **状态回溯 (Undo/Redo)**：记录操作历史。
    *   **深度优先搜索 (DFS)**：迷宫寻路、树遍历。
*   **队列 (FIFO)**：
    *   **消息队列**：网络包处理、事件分发。
    *   **广度优先搜索 (BFS)**：寻找最短路径、波纹扩散效果。
    *   **对象池 (Object Pool)**：空闲对象入队，借出对象出队。

## 4. 树 (Tree) 与 图 (Graph)

### 4.1 四叉树 (QuadTree) / 八叉树 (Octree)
*   **核心思想**：空间划分。将 2D/3D 空间递归分割为 4/8 个子区域。
*   **Unity 实战**：
    *   **场景管理**：只渲染摄像机视野内的物体（视锥体剔除）。
    *   **碰撞检测**：只检测同一区域内的物体碰撞，避免 O(N^2) 全局检测。
    *   **地形 LOD**：远处的地形块合并渲染，近处细分。

### 4.2 堆 (Heap) / 优先队列
*   **核心思想**：完全二叉树，父节点总是大于（或小于）子节点。
*   **实战：A* 寻路算法**
    *   **Open List**：使用最小堆存储待探索节点，每次取 `F` 值（代价+估价）最小的节点。
    *   **优化**：C# 默认没有 `PriorityQueue`（直到 .NET 6）。在 Unity 中通常手写一个二叉堆，或使用 `SortedDictionary`（性能较差）。

## 5. 高频算法面试题 (Unity 向)

### 5.1 快速排序 (QuickSort)
*   **原理**：分治法。选基准值 (Pivot)，比它小的放左边，比它大的放右边。
*   **平均复杂度**：O(N log N)。
*   **最坏复杂度**：O(N^2)（已排序数组）。
*   **Unity 应用**：`List.Sort()` 底层是内省排序 (Introsort)，混合了快排、堆排和插排。

### 5.2 二分查找 (Binary Search)
*   **前提**：**有序数组**。
*   **复杂度**：O(log N)。
*   **Unity 应用**：
    *   在有序的时间轴关键帧中查找当前帧。
    *   在权重表中查找随机掉落物品（累加概率 + 二分）。

### 5.3 动态规划 (Dynamic Programming)
*   **核心**：状态转移方程 + 缓存中间结果 (Memoization)。
*   **实战：背包问题**
    *   **场景**：玩家背包容量有限，每个道具占不同格子且有不同价值，如何装价值最大？
    *   **思路**：`dp[i][w]` 表示前 i 个物品放入容量 w 的最大价值。

### 5.4 几何算法
1.  **点在多边形内**：射线法。从点向任意方向发射射线，与边交点为奇数则在内，偶数则在外。
2.  **向量点积 (Dot Product)**：
    *   `Dot(A, B) > 0`：方向基本一致（夹角 < 90度）。
    *   `Dot(A, B) = 0`：垂直。
    *   **应用**：判断怪物在玩家前方还是后方。
3.  **向量叉积 (Cross Product)**：
    *   结果垂直于 A 和 B 构成的平面。
    *   **应用**：判断点在向量的左侧还是右侧（用于赛车游戏判断转弯方向）。
