# Addressables排障与优化手册（实战版）

## 1. 目标
- 出问题时知道先看哪里，而不是盲目重打包
- 建立“现象 -> 原因 -> 验证 -> 处理 -> 预防”的排障闭环

## 2. 常见故障地图

## 2.1 初始化失败
- 典型现象：
  - 初始化超时
  - 目录下载失败
  - 首次进入卡住
- 常见原因：
  - LoadPath配置错误
  - CDN证书/跨域/鉴权问题
  - Catalog路径和版本不一致
- 快速验证：
  - 检查运行时实际URL
  - 检查Catalog请求是否200
  - 本地切回离线Catalog排除网络问题

## 2.2 资源加载失败（Key not found / InvalidKey）
- 典型现象：地址存在但运行时报找不到
- 常见原因：
  - 地址改名未同步
  - Label配置遗漏
  - 分环境Catalog不一致
- 快速验证：
  - 输出当前Catalog中可用键集合
  - 比对构建产物与运行环境版本号

## 2.3 下载成功但实例化失败
- 典型现象：Bundle已下载，Instantiate时报错
- 常见原因：
  - 依赖脚本缺失或类型变更
  - 资源引用断链
  - 平台差异资源格式不兼容
- 快速验证：
  - 同版本本地直连资源复现
  - 打开依赖分析确认所有依赖包存在

## 2.4 内存上涨不回落
- 典型现象：场景切换后内存持续上升
- 常见原因：
  - Handle未Release
  - 对象池长期持有资源引用
  - 事件订阅未退订间接保持引用
- 快速验证：
  - 统计加载/释放调用次数是否对称
  - 逐模块禁用定位泄漏源

## 2.5 更新后资源错乱
- 典型现象：旧资源和新资源混用、表现不一致
- 常见原因：
  - 缓存污染
  - 目录更新流程不完整
  - 内容状态文件使用错误
- 快速验证：
  - 清缓存后冷启动对比
  - 强制更新Catalog并打印版本签名

## 3. 排障流程标准化

## 3.1 第一步：界定问题类型
- 下载问题
- 解析问题
- 依赖问题
- 生命周期问题

## 3.2 第二步：抓关键日志
- 操作开始/结束时间
- 请求URL、状态码、耗时
- Key、Label、Group信息
- Handle引用计数轨迹

## 3.3 第三步：最小化复现
- 只保留一个资源链路
- 锁定单平台单网络环境
- 固定Catalog版本

## 3.4 第四步：修复与回归
- 修复后跑冷启动 + 热更新 + 断网恢复三套流程
- 同时验证内存与性能回归

## 4. 优化专题

## 4.1 Bundle粒度优化
- 过大Bundle：下载慢，更新成本高
- 过小Bundle：请求过多，管理复杂
- 建议：以“更新频率+业务模块”折中切分

## 4.2 重复依赖治理
- 问题：同一贴图/材质跨组重复打包
- 手段：
  - 建立公共依赖组
  - 统一入口资源引用
  - 定期做重复资源扫描

## 4.3 启动时长优化
- 把必需资源列入启动白名单
- 非关键资源懒加载
- 预下载阶段显示可交互进度反馈

## 4.4 弱网鲁棒性优化
- 下载重试：指数退避而非固定间隔
- 请求超时分层：Catalog短超时，Bundle可长超时
- 断点续传与失败分块重试

## 4.5 内存治理
- 场景退出统一释放临时句柄
- 对象池资产分层（常驻池 vs 临时池）
- 资源引用可视化追踪，避免“隐式持有”

## 5. 线上发布与回滚策略

## 5.1 发布前
- 预生产环境演练全量与增量更新
- 验证旧版本客户端升级路径
- 记录每次发布Catalog签名与包列表

## 5.2 发布中
- 先灰度小流量观察失败率与下载耗时
- 指标阈值触发自动熔断

## 5.3 发布后
- 监控关键指标：
  - 初始化失败率
  - 资源加载失败率
  - 平均下载时长
  - 崩溃率与内存峰值
- 异常时按预案回滚到上个稳定Catalog

## 6. 面试高分案例模板（可直接背）

## 6.1 模板结构
- 背景：为什么要做Addressables改造
- 问题：线上遇到什么失败模式
- 诊断：你如何定位根因
- 方案：架构调整与策略优化
- 结果：稳定性/时长/包体/内存改进数据
- 复盘：沉淀了什么规范

## 6.2 示例口述
- “我们曾遇到更新后Key偶发失效。排查发现是多环境Catalog混用。最终把Profile环境隔离、版本签名校验、启动阶段目录一致性检查做成标准流程，加载失败率显著下降。”

## 7. 自测清单（每周一次）
- [ ] 我能在5分钟内判断问题属于下载、解析、依赖或生命周期
- [ ] 我有统一日志字段，不靠临时打印
- [ ] 我有固定的弱网与断网回归流程
- [ ] 我有可执行的回滚开关而非人工临时改配置
- [ ] 我能说明最近一次优化的量化收益

## 8. 与图形学联动点（加分项）
- 资源分包策略直接影响贴图/材质加载时机
- 加载抖动会放大渲染首帧尖峰
- Addressables优化要和渲染预热、Shader Variant管理联合设计
